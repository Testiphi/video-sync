# FrameSync — 视频同帧同步对比

两个带计时器的视频，自动或手动标注计时器数值对齐两个视频，纯前端实现，无需任何后端。

## 🚀 在线使用（无需安装）

直接打开 GitHub Pages 即可使用：

👉 **https://testiphi.github.io/video-sync/**

所有处理在浏览器本地完成，不上传任何文件。

> ⚠️ **从 GitHub Pages 访问时，直接输入文件路径无法加载视频**（浏览器安全限制）。
> 请使用「📂 浏览」或拖放文件来加载视频。

## 📦 本地运行

```bash
# 克隆仓库
git clone https://github.com/Testiphi/video-sync.git
cd video-sync

# 任何 HTTP 服务器打开 index.html
python -m http.server 8765
# 浏览器打开 http://localhost:8765
```

## 使用流程

### ① 加载视频
- 点击「📂 浏览」选择视频文件，或直接拖放文件到面板
- 本地使用时也可输入视频文件路径后点击「加载」
- 浏览器本地处理，不上传任何文件

### ② 框选计时器区域
- 在画面上拖拽框选计时器出现的区域
- 使用滑块 / 🎲 随机跳转浏览不同画面
- 或点击「自动」设置默认区域（右上角 82%×8%）
- 选区以百分比坐标存储，与视频分辨率无关

### ③ 校准计时器（二选一）

**方式 A — Tesseract OCR 自动识别**
- 点击「🎲 采样5帧」生成随机采样画面
- 点击「🔍 Tesseract OCR」，自动识别每帧的计时器数值并填入
  - 首次使用需下载 ~4MB OCR 引擎数据（后续缓存）
- 自动计算稳健回归（支持异常值检测告警）

**方式 B — 手动标注**
- 播放视频 → 在合适位置暂停
- 在下方输入当前画面计时器数值（秒），点击「添加」
- 添加 2 个以上校准点（覆盖不同时间位置），自动计算线性回归
- 下方显示「计时器区域放大效果」辅助读取
- 显示回归公式 `视频时间 = a × 计时器 + b` 和拟合优度 R²

### ④ 同步对比
- 两个视频索引都构建完成后自动进入同步模式
- 拖动滑块控制目标计时器值 → 两个视频同时跳到对应位置
- B 视频偏移微调 ±0.1s / ±1s（含 ±0.01s 精细步长）

## 技术栈

- **纯静态 HTML / CSS / JS** — 一个 `index.html` 搞定所有
- **视频处理**: `<video>` 元素 + Canvas 截图
- **OCR 识别**: [Tesseract.js](https://tesseract.projectnaptha.com/)（浏览器端 WebAssembly，纯前端）
- **回归算法**: Theil-Sen 稳健线性回归 + 离群点检测告警

## 与先前版本的差异

| 功能 | 纯静态版 (v1) | 当前版 (v2) |
|------|---------------|-------------|
| OCR | 无 | Tesseract.js 浏览器端 |
| 回归 | 最小二乘法 (OLS) | Theil-Sen 稳健回归 |
| 异常值检测 | 无 | 残差 >1s 标红警告 |
| 偏移微调 | ±0.1s / ±1s | 新增 ±0.01s 步长 |
| 帧预览 | 异步加载 | 独立渲染消除竞态 |
| 同步范围 | 校准点区间 | 全视频外延 |
