# FrameSync — 双视频同步播放器

两个带计时器的视频，手动标注计时器数值对齐两个视频，纯前端实现，无需任何后端。

## 🚀 在线使用（无需安装）

直接打开 GitHub Pages 即可使用：

👉 **https://testiphi.github.io/video-sync/**

所有处理在浏览器本地完成，不上传任何文件。

## 📦 本地运行

```bash
# 克隆仓库
git clone https://github.com/Testiphi/video-sync.git
cd video-sync

# 任何 HTTP 服务器打开 index.html
python -m http.server 8765
# 浏览器打开 http://localhost:8765
```

也可以直接用浏览器双击打开（部分浏览器需要 HTTP 服务器才能加载视频）。

## 使用流程

![工作流](docs/workflow.png)

### ① 加载视频
- 点击「选择视频」分别加载视频 A 和 B
- 浏览器本地处理，不上传任何文件

### ② 框选计时器区域
- 点击「框选计时器区域」→ 暂停视频 → 在画面上拖拽框选计时器出现的区域
- 或点击「推荐位置」快速设置默认区域（右上角）
- 选区以百分比坐标存储，与视频分辨率无关

### ③ 校准计时器
- 在时间输入框输入秒数，点击「跳转」定位到目标画面
- 下方「计时器区域放大效果」会实时显示 ROI 裁剪画面
- 查看放大画面，读取计时器数值，输入后点击「添加」
- **添加 2-3 个校准点**（覆盖不同时间位置），自动计算线性回归
- 显示回归公式 `视频时间 = a × 计时器 + b` 和拟合优度 R²

### ④ 同步播放
- 回归计算完成后自动进入同步模式
- 拖动滑块控制目标计时器值 → 两个视频同时跳到对应位置
- 支持 A/B 独立微调 ±0.1s / ±1s

## 技术栈

- **纯静态 HTML / CSS / JS** — 一个 `index.html` 搞定所有
- **视频处理**: `<video>` 元素 + Canvas 截图 + `URL.createObjectURL()`
- **回归算法**: 最小二乘法线性回归（纯 JS 实现）
- **持久化**: localStorage 自动保存校准数据

## 数据存储

所有校准数据（ROI 选区、校准点、回归参数）自动保存在浏览器 localStorage 中：
- **手动保存**: 点击底部「保存数据」
- **自动保存**: 每 30 秒自动保存一次
- **恢复数据**: 重新打开页面后点击「恢复数据」
- 视频文件需要重新加载（浏览器安全限制），但校准数据保留
